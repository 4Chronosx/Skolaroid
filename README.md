## 🗺️ Project Overview

**Skolaroid** is a **location-based memory sharing platform** designed specifically for university campuses. It allows students to **capture, geo-tag, and share meaningful moments** tied to specific campus buildings and locations, creating a living digital archive of campus life.

The platform transforms how students document and discover their campus experience by combining **interactive maps**, **location-based memories**, and **social features** into a cohesive community-driven platform. Whether it's a first-day photo at the library, a study session at the science building, or a memorable event at the student center, Skolaroid helps preserve these moments forever.

### 💡 Why Skolaroid?

Campus life is filled with meaningful moments that deserve to be remembered and shared. While generic social media platforms exist, they lack the contextual connection to physical spaces that make university experiences unique. Skolaroid bridges this gap by creating a dedicated space for campus memories.

- 📍 **Location-Based Memories:** Attach photos and stories to specific campus buildings, creating a rich tapestry of experiences tied to physical locations.
- 🗺️ **Interactive Campus Map:** Explore an interactive Mapbox-powered map showing all campus buildings with their associated memories and stories.
- 🏷️ **Smart Tagging & Discovery:** Organize memories with tags and categories, making it easy to discover moments by theme, event, or location.
- 👥 **Community Engagement:** Connect with fellow students through shared experiences, reactions, and collaborative memory collections.

## 💻 Getting Started

Follow these steps to set up and run **Skolaroid** locally.

### 🔧 Prerequisites

- [Node.js](https://nodejs.org/) (version 20.0 or higher)
- [pnpm](https://pnpm.io/)
- [PostgreSQL](https://www.postgresql.org/) (or use Supabase)
- [Supabase Account](https://supabase.com/) (for auth and database hosting)
- [Mapbox Account](https://www.mapbox.com/) (for map features)

### 🛠️ Installation

#### 1. Clone the Repository

```sh
git clone https://github.com/4Chronosx/Skolaroid.git
cd Skolaroid
```

#### 2. Install Dependencies

```sh
pnpm install
```

#### 3. Environment Setup

Create a `.env` file in the root directory with the following variables:

```bash
# Database
DATABASE_URL="postgresql://user:password@host:5432/skolaroid"
DIRECT_URL="postgresql://user:password@host:5432/skolaroid"

# Supabase
NEXT_PUBLIC_SUPABASE_URL="your-supabase-url"
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_DEFAULT_KEY="your-supabase-anon-key"

# Mapbox
NEXT_PUBLIC_MAPBOX_TOKEN="your-mapbox-token"
```

#### 4. Run Database Migrations

```sh
npx prisma migrate dev
npx prisma db seed  # Optional: seed with sample data
```

### ▶️ Running the Application

#### Development Server

```sh
pnpm dev
```

The application will be available at `http://localhost:3000`

#### Production Build

```sh
pnpm build
pnpm start
```

#### Type Checking & Linting

```sh
pnpm type-check
pnpm lint
pnpm format
```

├── src/

│   │   │   └── prisma/       # Prisma-based endpoints
│   │   ├── auth/             # Authentication pages
│   │   ├── memories/         # Memory pages
│   │   ├── profile/          # User profile
│   │   └── page.tsx          # Landing page

│   │   ├── supabase/         # Supabase clients & auth
